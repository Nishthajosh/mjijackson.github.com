Ruby Config Objects Revisited

Thursday, 08 Apr 2010
#ruby

Since writing about Ruby configuration objects a few months ago, I’ve done quite a bit of hacking on my original Config class. My main issue with it is that it’s basically serving as a wrapper for a Hash, but it’s a foreign interface. In other words, if you hand a Config object to another developer, he’s going to have to look up the API before he can use it. But that defeats the purpose for writing it in the first place.
